# 服务间通信与权限验证

## 主题目标

建立统一的服务身份认证、方法授权和用户上下文透传基线，确保每一跳都能独立证明身份并按本地规则授权。

## 必须遵守

- 业务参数进入 proto request
- 服务身份和用户上下文进入 gRPC metadata
- 当前统一字段名为 `x-service-token`、`x-user-id`、`x-user-authenticated`
- 调用方为每一跳签发短时效服务令牌
- metadata 由统一 client、入口适配层或 interceptor 注入，不由业务代码手工拼装
- 被调用方独立验签并按本地策略表做方法授权
- 业务 handler 只读取注入到 context 的可信身份，不直接读取原始 metadata
- 上下游之间逐跳签发、逐跳授权、逐跳最小透传

## 常见错误

- 把服务令牌或用户字段放进业务 request
- 在业务代码里自行拼装 `x-service-token`
- 在完成服务认证前直接信任 `x-user-id`
- 把上游签发的服务令牌原样转发给下游
- 直接在 handler 中读取原始 metadata 做业务判断

## 落地检查点

- 服务间调用是否总是通过统一 interceptor 或 client 注入 metadata
- 缺失或验签失败时是否返回明确的未认证错误
- 方法授权是否归被调用方本地策略决定
- 用户字段是否只在确实源于用户请求时才最小透传
- 多跳链路中是否每一跳都重新签发本跳服务令牌
