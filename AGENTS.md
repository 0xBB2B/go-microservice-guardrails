# Go Microservice Guardrails

本仓库提供一个可按需加载的 Skill：

- `go-microservice-guardrails` -> `skills/go-microservice-guardrails/SKILL.md`

当用户的任务与以下任一主题相关时，你必须先读取并遵循对应的 `SKILL.md`：

- 使用 Go 设计、生成、整改或评审微服务架构
- 设计或评审 `contracts/`、`pkg/`、`services/`、`ops/` 的仓库分层
- 设计或评审 gRPC、proto、Buf、共享契约与代码生成
- 设计或评审服务边界、接口所有权、配置边界
- 设计或评审服务间鉴权、用户上下文透传、方法授权与最小权限数据访问
- 设计或评审可观测性、缓存刷新、热更新、部署与测试分层

使用要求：

1. 先判断当前任务属于“生成”“整改”“评审”中的哪一类。
2. 先处理边界与所有权，再处理契约与通信，最后再进入实现细节。
3. `规范/` 目录下的文件是规则源文件，`go-microservice-guardrails` Skill 基于这些源文件生成；对外输出时不要直接引用这些源文件。
4. 如需校验或更新 Skill，可按需读取 `规范/` 下与当前主题相关的源文件，但不要一次性把所有规范都加载进上下文。
5. 如果当前上下文不足以判断边界是否正确，必须明确列为待确认项，不能把猜测写成既成事实。
